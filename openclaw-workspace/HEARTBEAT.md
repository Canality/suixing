# HEARTBEAT · 7x24 后台任务

## 心跳执行频次
每60秒运行一次心跳检测，评估是否需要触发动作。

## 监控任务

### 1. 餐食时段监控 (meal_monitor)
**触发条件**:
- 7:00-8:30 (早餐窗口)
- 11:30-13:00 (午餐窗口) 
- 17:30-19:30 (晚餐窗口)
- 21:00-22:00 (夜宵窗口)

**动作**:
1. 检测用户当前状态(是否已触发过)
2. 搜索附近餐厅(根据时段+用户口味)
3. 检查是否有排队/优惠信息
4. 推送推荐卡片(每时段最多1次)

**状态机**: `idle → recommended → decided → queueing → completed`
**防打扰**: 用户正在聊天时跳过，等待下次心跳

### 2. 排队监控 (queue_monitor)
**触发条件**: 用户有活跃的排队取号(状态=queueing)

**动作**:
1. 每30秒查询排队进度
2. 剩10桌时 → 推送温和提醒
3. 剩5桌时 → 紧急提醒 + 自动查询打车路线
4. 过号时 → 提醒重新取号

**连锁示例**:
```
排队剩5桌 → 推送"现在出发刚好！"
  → 自动查询: 打车 望京→餐厅 ¥18, 12分钟
  → 推送"已帮你预估打车，预计18:27到达"
```

### 3. 天气+活动监控 (weather_activity_monitor)
**触发条件**:
- 每天早上7:00(当日天气)
- 周末早上9:00(周末活动推荐)
- 天气突变时(如下雨→推荐室内活动)

**动作**:
1. 获取最新天气
2. 搜索当天/周末活动
3. 好天气 → 推荐户外活动+单车路线
4. 坏天气 → 推荐室内活动+外卖优惠
