# 评委验收测试用例

> 分为两大类：**🎭 演示剧本**（展示主动智能亮点，适合现场演示）和 **📋 系统用例**（覆盖所有基础功能，适合逐项评分）。
> 满分 **100 分**。

---

## 🎭 演示剧本 (三大核心创新场景，30 分)

这三个剧本展示了随行区别于普通 chatbot 的核心能力 —— **主动感知、跨 Skill 协同、后台自主决策**。

---

### 剧本一：从"想吃"到"出发"的主动协同 (10 分)

> 验证: 餐饮 → 排队监控 → 跨 Skill 联动出行

**演示步骤:**

| 步 | 操作 | 预期 |
|----|------|------|
| 1 | 输入: **"帮我看看望京附近的火锅店"** | 返回 2+ 家火锅店，标注 `[美团]`，显示实时排队 |
| 2 | 输入: **"帮我在后院火锅取个号"** | 取号成功，返回排队号码 + 预计等待 |
| 3 | 观察右侧 SSE 面板 | 在输入第 2 步时，Agent 自动调用了 `create_watch` 创建排队监控（事后可访问 `/api/watch/list` 验证） |
| 4 | 等待沙盒更新或访问 `/api/watch/list` | 确认后台监控任务已创建，条件为"排队桌数 ≤ 阈值" |

**评分要点:**

- [ ] Agent 返回了实时排队数据 (3 分)
- [ ] 取号成功，数据完整 (2 分)
- [ ] **关键**: Agent 自动创建了后台监控，而非被动等待用户说"有位置了叫我" (3 分)
- [ ] 监控任务可通过 API 验证 (2 分)

**亮点解读**: 普通 chatbot 取完号就结束了。随行会在后台持续监控排队进度，快到时主动推送提醒并联动叫车 —— 这是 **watchdog 长时监控 + 跨 Skill 协同** 的核心能力。

---

### 剧本二：环境突变下的管家关怀 (10 分)

> 验证: 记忆系统 + 环境变化主动通知

**演示步骤:**

| 步 | 操作 | 预期 |
|----|------|------|
| 1 | 输入: **"帮我记住: 我喜欢骑行，周六下午习惯去温榆河，预算 200 以内"** | Agent 调用 `remember` 记录多项偏好 |
| 2 | 输入: **"明早想去奥森公园骑行"** | Agent 查天气 → 确认天气好 → 推荐路线 + 生成高德导航链接 → **自动创建天气反向监控** |
| 3 | 等待沙盒天气变化 (或观察 SSE 面板的 `heartbeat_weather` 事件) | 当天气变差时，右侧 SSE 面板显示 `watch_triggered` 事件 |
| 4 | 观察 Agent 主动推送 | 推送消息应当: 告知天气变化 + 建议室内替代方案 + 引用之前记忆的偏好 |

**评分要点:**

- [ ] Agent 正确记录了用户的多项偏好 (2 分)
- [ ] 第 2 步返回了路线规划 + 导航链接 (2 分)
- [ ] **关键**: 天气好时也创建了反向监控("万一天气变差就通知") (3 分)
- [ ] 主动推送消息引用了用户记忆中的偏好 (3 分)

**亮点解读**: 管家不是"你问天气好就推荐"，而是"天气好我推荐，同时帮你盯着天气——万一下雨我第一个通知你，还帮你找好室内备选方案"。这才是 **7x24 私人管家** 的感觉。

---

### 剧本三：机遇捕捉 —— 有人退票了 (10 分)

> 验证: 机遇事件分类 + 稀有度节流 + 即时推送

**演示步骤:**

| 步 | 操作 | 预期 |
|----|------|------|
| 1 | 输入: **"想看脱口秀开放麦，但好像没票了"** | Agent 查活动状态 → 确认 sold_out → **自动创建票源监控** → 回复"已帮你盯着，有人退票立刻通知" |
| 2 | 观察左侧沙盒事件面板 (左上角 🎲) | 沙盒每 5 秒刷新，显示随机事件 |
| 3 | 当沙盒触发 `ticket_released` 事件时 | 右侧 SSE 面板立即显示 `watch_triggered` |
| 4 | Agent 主动推送 | 推送消息: "小明，刚才有人退了 1 张脱口秀的票！[立即购买]" |

**评分要点:**

- [ ] Agent 发现售罄后自动创建了票源监控 (3 分)
- [ ] 未骚扰用户——只在条件满足时推送一次 (2 分)
- [ ] **关键**: 沙盒事件面板显示了动态事件 → SSE 面板显示了触发 → Agent 主动推送（完整链路可观察）(3 分)
- [ ] 推送消息包含具体行动指引 (2 分)

**亮点解读**: 这是 **机遇事件 + 稀有度节流** 的完美展示。管家不会在"没票"时就推一堆替代推荐骚扰你，而是安静地在后台等待那个 5% 概率的退票事件，一旦触发立即通知。这就是"全天候私人管家"的核心价值。

## 📋 系统功能用例 (70 分)

### TC-01: 餐厅搜索 (10 分)

**输入:** `帮我推荐望京附近的火锅，要排队少的`

| 检查项 | 预期 |
|--------|------|
| 调用工具 | `search_restaurants` (area=望京, cuisine=火锅) |
| 返回结果 | 列出 ≥2 家火锅店，含海底捞、后院火锅等 |
| 动态数据 | 显示实时排队人数、等待时间 |
| 来源标签 | `[美团]` |
| 排序 | 综合评分 + 排队情况排序 |

---

### TC-02: 排队查询 + 取号 (10 分)

**第1轮:** `海底捞火锅现在排队要等多久`
**第2轮:** `帮我取个号`

| 检查项 | 预期 |
|--------|------|
| 第1轮 | `check_queue` 返回排队状态 |
| 第2轮 | 取号成功，返回排队号码 + 前面桌数 + 预计等待 |

---

### TC-03: 多方式路线规划 (10 分)

**输入:** `从望京SOHO到798艺术区怎么走`

| 检查项 | 预期 |
|--------|------|
| 调用工具 | `plan_route` + `generate_route_link` |
| 返回 ≥2 种方式 | 打车 + 单车，含距离/时间/费用 |
| 来源标签 | `[高德]` |
| 导航链接 | 包含可点击的高德导航 URL |

---

### TC-04: 天气查询 + 活动联动 (10 分)

**输入:** `今天天气怎么样，适合出去玩吗`

| 检查项 | 预期 |
|--------|------|
| 天气数据 | 温度 + 天气状况 + 湿度 + 风力 + AQI |
| 活动建议 | 根据天气给出室内/室外建议 |
| 来源标签 | `[天气网]` |

---

### TC-05: 周末活动推荐 (10 分)

**输入:** `周末有什么好玩的`

| 检查项 | 预期 |
|--------|------|
| 多工具联动 | `get_weather` + `search_activities` |
| 活动覆盖 | ≥3 个活动，含电影/展览/户外/演出等 |
| 来源标签 | 电影 `[猫眼]`、展览 `[大麦]` 等 |
| 天气联动 | 根据天气推荐室内/室外优先级 |

---

### TC-06: 用户记忆存储 + 个性化推荐 (10 分)

**第1轮:** `记住我喜欢吃川菜和日料，人均预算不超过100元，家在望京`
